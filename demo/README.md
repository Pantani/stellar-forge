# demo

Generated by `stellar forge init`.

## Next steps

1. Run `stellar forge doctor`.
2. Run `stellar forge project validate`.
3. Run `stellar forge dev up`.
4. Build and deploy contracts with `stellar forge contract build` and `stellar forge release deploy testnet`.
5. Export runtime values with `stellar forge release env export testnet` when you are ready to wire apps to a real deploy.

## Daily loop

```bash
stellar forge doctor
stellar forge project validate
stellar forge project sync
stellar forge dev up
stellar forge dev reseed --network testnet
stellar forge release plan testnet
```

## Repo helpers

- `node scripts/doctor.mjs`

- `node scripts/release.mjs --plan`
- `node workers/events/ingest-events.mjs <resource> --status --once`

## Generated layout

- `stellarforge.toml`: desired project state
- `stellarforge.lock.json`: observed deployment state
- `.env.example`: manifest-derived defaults
- `.env.generated`: runtime values exported from actual state
- `dist/`: deploy artifacts, fetched Wasm files, and snapshots
- `scripts/`: small wrappers for doctor, reseed, and release flows
- `workers/events/`: event helper scripts and cursor snapshot
- `apps/api`: generated API scaffold and OpenAPI output
- `apps/web`: generated frontend scaffold and smoke runners

## Run generated apps

### API

- `pnpm --dir apps/api install`
- `pnpm --dir apps/api dev`

### Frontend

- `pnpm --dir apps/web install`
- `pnpm --dir apps/web dev`
- `pnpm --dir apps/web smoke:ui`

## Frontend validation

- `stellar forge project smoke`
- `stellar forge project smoke --browser`
- `pnpm --dir apps/web smoke:browser:build`
- `pnpm --dir apps/web smoke:browser:install`
- `pnpm --dir apps/web smoke:browser:run`

The full browser smoke skips the Chromium install step when the pinned Playwright browser bundle is already cached. If port `4173` is already in use, rerun with `STELLAR_FORGE_BROWSER_SMOKE_PORT=<port>`.
